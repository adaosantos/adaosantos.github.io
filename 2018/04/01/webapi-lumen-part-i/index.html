    O Lumen é um microframework criado a partir do Laravel, que é a menina dos olhos da galera que desenvolve PHP ultimamente.

    Esse artigo não vai falar sobre os conceitos básicos do Laravel ou do Lumen, para tal eu recomendo a leitura da documentação https://lumen.laravel.com/docs/5.6#installation

    Autenticação com Lumen

    Quando criamos um projeto novo com Lumen ele já vem um alguns itens que são necessários para um solução completa de autenticação
    Vamos dar uma olhada neles:

    Estes arquivos contém uma lógica primitiva de autenticação, vamos analisar primeiro o AuthService, ele é responsável pela por conter a lógica que vai recuperar o usuário authenticado na requisição.

    No método boot é chamada a instância de auth (que é um singleton) onde registramos a função anônima no método viaRequest, basicamente esse método recebe um request e tenta encontrar o usuário com base em um token, até aqui nada de especial.

    Agora vejamos o middleware de autenticação Authenticate:

    Dentro dele, em seu construtor, nós temos a instância da classe Auth, ela é interface que provê meios de recuperar o usuário, ela representa a instância $this->app['auth']que chamamos na AuthServiceProvider. Basicamente esse middleware verifica se o provider consegue recuperar algum usuário, caso não retorna Unauthorized.

    Você pode ler mais sobre como essa mágica acontece na documentação do Lumen (clique aqui)
